REACT Native

What is it and what is it eaten with?

ReactNative - is a JavaScript-based mobile app platform that allows you to build cross-platform mobile apps.

Some history

2012 - facebook confession: "the biggest mistake we made as a company was that we relied too much on HTML, not native".

2013 - Dan Walk made a revolutionary discovery - he found a method for creating user interface elements for iOS applications using JavaScript.

2015 - Facebook officially released React Native

VS

React.js - is a JavaScript library that helps companies create beautiful UIs. One of the main features of React is that it can run on the client side and render on the server side, and these parts can interact with each other.

React Native - is a mobile app development framework that builds mobile apps using React.js. It allows you to create a rich mobile user interface from declarative components. React Native allows you to develop applications for Android, iOS and Windows at the same time.

React.js uses Virtual DOM, while React Native uses native APIs.

- REACT Native is a great choice for developers with JavaScript background, as there is no need to learn Java for Android or Swift for iOS.

- REACT Native has gained a lot of popularity in recent years, the reason is that it helps to serve both platforms at the same time and end the holy war of choice for iOS or Android development for mobile developers.

How it's work?

There are two important threads running in every React Native app:

- the main thread, which also runs in every standard native application;

- specific to React Native. Its job is to execute JavaScript code in a separate JavaScript engine.

How do the threads interact?

Between these two threads is the so-called bridge, which is the core of React Native. The bridge has three important characteristics:

  • Asynchronous
  • Batched
  • Serializable
